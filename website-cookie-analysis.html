Website Cookie Analysis - December 2024

Problem

How can businesses utilize website cookie data to better understand, target, and extract revenue from their online customer base? This project explores user behavior patterns, revenue drivers, and predictors of transaction activity using cookie-level data.

Dataset Overview

We used anonymized cookie-level web data from an online store, spanning from August 1, 2010 to January 31, 2011. The dataset contains over 16,000 records, each representing a "website session" where user activity was tracked. A cookie is a packet of data used to identify users and monitor how they navigate the site across sessions. The data includes variables such as bounces, time on site, number of visits, revenue, and traffic source.

Exploration & Segmentation

Users originated from 3,570 different cities and 697 regions, with top cities including New York, Moscow, Kyiv, and London. Engagement levels were segmented based on time spent on site: low (0–28 min), medium (29–87 min), and high (87+ min). Interestingly, referral sources were consistently the lowest driver of revenue across all segments.

We also observed seasonal patterns typical of ecommerce behavior—spikes in site visits, revenue, and session time leading up to the holiday season (November–December), followed by a sharp drop in January.

Search Engines & Revenue

We analyzed organic traffic sources using a multiple linear regression model. Visits from Google stood out with a strong positive correlation to revenue—each Google visit generated an estimated $9.37, holding all other variables constant. Overall, each additional visit drove approximately $15.86 in revenue, while bounces resulted in a revenue loss of $19.55. These findings reinforce the importance of Google-based SEO strategies.

Predicting Transactions

We developed both logistic regression and random forest models to predict whether a user would complete a transaction. Logistic regression was ultimately chosen for its simplicity and interpretability, delivering an accuracy of 89.81%. Significant predictors included the number of visits (positive), new visitor status (positive), and bounce behavior (strongly negative). Organic traffic users were found to be 1.5x more likely to transact than those from other channels.

Scoring & Targeting

Using predicted transaction probabilities, we developed two targeting strategies. The first, a limited supply rule, allows the business to achieve 100 transactions by targeting just the top 102 users ranked by transaction probability. The second, a marginal supply rule, targets the top 953 users, maintaining a transaction rate above 50%. These scoring approaches enable precise campaign targeting, optimizing outreach efforts and budget allocation.

Business Insights

Our findings point to several strategic recommendations. First, businesses should double down on SEO strategies with a focus on Google—its organic traffic generates the highest return. Second, re-engagement campaigns that drive more return visits from previously engaged users can boost revenue. Retargeting, email follow-ups, and personalized promotions are key tools here.

Third, user experience should be optimized to reduce bounce rates, which are closely tied to lost revenue. We recommend mobile-first design, clear navigation, faster page loads, and real-time customer support via chatbots or live agents. Together, these changes can improve conversion efficiency and unlock greater value from website traffic.
